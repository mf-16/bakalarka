Python stores default namespace in property NamespaceManager#default_ns_uri

When serializing PROVN format, Python uses custom [ProvNSerializer](https://github.com/trungdong/prov/blob/master/src/prov/serializers/provn.py).
When serializing default namespace im method [ProvBundle#get_provn()](https://github.com/trungdong/prov/blob/1cb710a26c623c5c706b1c6945c645efd9e925c0/src/prov/model.py#L1431), on line 1444:

\`\`\`
default_namespace = self._namespaces.get_default_namespace()
    if default_namespace:
        lines.append("default <%s>" % default_namespace.uri)
\`\`\`

gets the default_namespace and appends it to lines.

At the end on line 1466:

\`provn_str = newline.join(lines) + "\n"\`

all the lines are joined.

When deserializing PROVN format, Java uses deserializer which uses parser generated by [ANTLR](https://www.antlr.org/) with this [grammar](https://github.com/lucmoreau/ProvToolbox/blob/master/modules-core/prov-n/src/main/antlr3/org/openprovenance/prov/notation/PROV_N.g).
The parser reads the default namespace without issues. When converting default namespace, Java calls method [Namespace#registerDefault()](https://github.com/lucmoreau/ProvToolbox/blob/ca05849c88a84aae736355569a8bdae78c68b072/modules-core/prov-model/src/main/java/org/openprovenance/prov/model/Namespace.java#L120), which just calls [Namespace#register()](https://github.com/lucmoreau/ProvToolbox/blob/ca05849c88a84aae736355569a8bdae78c68b072/modules-core/prov-model/src/main/java/org/openprovenance/prov/model/Namespace.java#L124) with null prefix:

\`register(null,namespace);\`

Namespace#register fills defaultNamespace field, if prefix is null on line 127:

\`\`\`
 if ((prefix == null) || (prefix.equals(XMLConstants.DEFAULT_NS_PREFIX))) {
    if (defaultNamespace == null) {
        defaultNamespace = namespace;
    } else {
        newPrefix(namespace);
    }
}
\`\`\`

So the default namespace is preserved like it should.
