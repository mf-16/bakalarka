Prov Python represents the prov:startTime and prov:endTime attributes as datetime.datetime(https://docs.python.org/3/library/datetime.html#datetime-objects)
Because when creating activity it uses the [_ensure_datetime()](https://github.com/trungdong/prov/blob/1cb710a26c623c5c706b1c6945c645efd9e925c0/src/prov/model.py#L34) function, which uses [dateutil.parser](https://dateutil.readthedocs.io/en/stable/parser.html). This parser returns datetime.datetime object.
Therefore, we have no problem creating a document with the timezones we want in memory.

Prov Python uses a custom [ProvNSerializer](https://github.com/trungdong/prov/blob/master/src/prov/serializers/provn.py) when serializing to PROV-N.
When serializing the prov:startTime and prov:endTime attributes in [ProvRecord#get_provn()](https://github.com/trungdong/prov/blob/1cb710a26c623c5c706b1c6945c645efd9e925c0/src/prov/model.py#L481) method on line 506:
\`\`\`
items.append(
    value.isoformat()
    if isinstance(value, datetime.datetime)
    else str(value)
)
\`\`\`
It uses the [datetime#isoformat()](https://docs.python.org/3/library/datetime.html#datetime.datetime.isoformat) method, which has no timezone loss. So, Prov Python serializes the document with the correct timezones.

When deserializing from PROV-N, ProvToolbox uses deserializer which uses parser generated by [ANTLR](https://www.antlr.org/) with this [grammar](https://github.com/lucmoreau/ProvToolbox/blob/master/modules-core/prov-n/src/main/antlr3/org/openprovenance/prov/notation/PROV_N.g).
The parser reads the timezone correctly, but when converting the string to the XMLGregorianCalendar, the timezone is lost.
More specifically here in method [ProvFactory#newISOTime()](https://github.com/lucmoreau/ProvToolbox/blob/974208486c9009f6313e1c21d462720ef72a7acd/modules-core/prov-model/src/main/java/org/openprovenance/prov/model/ProvFactory.java#L751), which uses a method [ProvFactory#newTime()](https://github.com/lucmoreau/ProvToolbox/blob/974208486c9009f6313e1c21d462720ef72a7acd/modules-core/prov-model/src/main/java/org/openprovenance/prov/model/ProvFactory.java#L929). On line 930 new GregorianCalendar object is created, and default system timezone is used.

\`GregorianCalendar gc = new GregorianCalendar();\`

After this, on line 931, only time is set.

\`gc.setTime(date);\`

Therefore, information about the timezone is lost.

This issue was introduced [here](https://github.com/lucmoreau/ProvToolbox/issues/209) and it was solved in this [commit](https://github.com/lucmoreau/ProvToolbox/commit/a1cb22cbddc60cd4563512f66f03caf58f9f068b) by adding a method where timezone is set as well.
